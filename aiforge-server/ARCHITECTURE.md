### aiforge 后端架构说明文档 (v1.0)

------

#### 1. 项目概述

`aiforge` 是一个基于 **Spring Boot 3** 和 **Java 21** 构建的 AI 开发平台。采用 **Maven 多模块 (Multi-Module)** 架构设计，旨在实现高内聚、低耦合，支持 AI 大模型集成、数据集交易及文章系统。

#### 2. 技术栈

- **核心框架**：Spring Boot 3.5.x
- **开发语言**：Java 21 (利用新特性如虚拟线程、Record 等)
- **持久层**：MyBatis-Plus (高效数据库操作)
- **数据库**：MySQL 8.0+
- **工具类**：Lombok, Hutool
- **依赖管理**：Maven

#### 3. 模块职责划分

| **模块名**         | **角色** | **核心职责**                                                 | **依赖关系**            |
| ------------------ | -------- | ------------------------------------------------------------ | ----------------------- |
| **aiforge-server** | 父工程   | 统一管理依赖版本、插件、模块聚合                             | -                       |
| **aiforge-common** | 公共模块 | 定义全局状态码、统一返回结果 (`Result`)、全局异常处理、工具类、基础实体类 | 无                      |
| **aiforge-system** | 系统模块 | 用户管理、角色权限 (RBAC)、日志审计、登录认证                | 依赖 `common`           |
| **aiforge-biz**    | 业务模块 | 数据集管理、文章交易逻辑、订单系统                           | 依赖 `common`, `system` |
| **aiforge-ai**     | AI 模块  | 大模型 API 对接 (Spring AI/LangChain4j)、RAG 检索增强、Prompt 模板 | 依赖 `common`           |
| **aiforge-web**    | 启动模块 | 全局 YAML 配置、主启动类、对外暴露的 Controller、拦截器配置  | 依赖所有业务模块        |

#### 4. 依赖流向原则

为了防止循环依赖，必须遵循以下拓扑顺序：

```
aiforge-web` ➔ `业务模块(biz/ai/system)` ➔ `aiforge-common
```

> **禁忌**：禁止 `common` 依赖任何其他子模块；禁止业务模块之间产生双向循环依赖。

#### 5. 关键规范

- **包名规范**：统一使用 `com.aiforge.[module]` 作为根包。
- **配置文件**：所有环境配置（Dev/Prod）统一定义在 `aiforge-web` 的 `resources` 下。
- **打包插件**：仅在 `aiforge-web` 中保留 `spring-boot-maven-plugin`，其余模块打包为普通 Jar。
- **异常处理**：业务异常在子模块抛出，由 `common` 模块的 `@RestControllerAdvice` 统一捕获。
